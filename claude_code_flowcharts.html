2. Startup Routing and Host Assembly

The real startup path is not "enter CLI and call the model". The entrypoint first handles fast-path routing and initialization, then decides whether execution enters the interactive host, the headless host, or a specific subcommand. REPL UI and QueryEngine are two host shells here, not a parent-child chain.

5. Session / Transcript / Memory / Compaction Loop

This replaces the old over-simplified "load/save history" framing. More accurately: Session is a recoverable runtime unit, Transcript is the message chain that can still feed query(), SessionMemory is a background-throttled long-term fact asset, and Compaction is part of the context-governance loop that feeds back into future recovery and cache behavior.

7. Context Governance and Compression

Context governance is not "summarize once when the chat is too long". It is a layered budget and compression system: first tool-result budgeting, then snip, microcompact, collapse, and only then autocompact; if needed, the system may also enter reactive compact after an error.
flowchart TD A[Begin context governance] --> B[tool result budget / content replacement] B --> C[snipCompactIfNeeded] C --> D[microcompact] D --> E[context collapse] E --> F[autocompact] F --> G[reactive compact when needed] G --> H[continue query loop]

8. Tool Concurrency Batching

Tool concurrency is also not "run all read-only tools at once". The system groups tools into safe batches based on isConcurrencySafe; concurrency-safe tools run within the same batch, while an unsafe tool becomes a serial barrier between batches.
